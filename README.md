## Approach

```
Human  →  仕様の解釈 / 要件の分解 / 品質の判断 / 次の問いを立てる
AI     →  実装 / テスト / リファクタリング / ドキュメント生成
```

「どう書くか」に時間を使うのではなく、**「何を作るか・なぜ作るか・どう評価するか」** に集中する開発スタイル。
公共系で培った厳格な品質基準を AI 運用に持ち込み、速度と信頼性を両立させることをテーマにしています。
